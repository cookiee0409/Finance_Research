================================================================
종목 라운지 — 데이터 출처 정리 (네이버증권 + 구글파이낸스 병행)
작성 기준: 종목 정보 1일 4회 갱신 (09:00 / 12:00 / 15:30 / 20:00 KST)
================================================================

[갱신 스케줄 — Vercel cron, UTC 기준]
  09:00 KST = 00:00 UTC   "0 0 * * *"   장 시작
  12:00 KST = 03:00 UTC   "0 3 * * *"   장중
  15:30 KST = 06:30 UTC   "30 6 * * *"  장 마감 직후
  20:00 KST = 11:00 UTC   "0 11 * * *"  마감 정리
  ※ 4회/일은 네이버·구글 모두 레이트리밋 부담 없음. 종목당 호출 간 200~400ms 딜레이 권장.


────────────────────────────────────────────────────────────
1. 구글파이낸스 (Google Finance)  — 사용 가능 여부 결론
────────────────────────────────────────────────────────────
• 공식 REST API 없음 (2012년 폐지).
• 페이지(google.com/finance) 직접 스크래핑: 클래스명 난독화 → 자주 깨짐. 비권장.
• 적법·안정 경로 = 구글 시트의 =GOOGLEFINANCE() + "웹에 게시(CSV)" → cron이 CSV fetch.
    - 구글이 의도한 사용 방식이라 ToS상 안전.
    - 준비물: 구글 계정으로 시트 1개 생성 → 셀에 =GOOGLEFINANCE(...) → 파일>공유>웹에 게시(CSV)
    - 예) =GOOGLEFINANCE("KRX:005930","price") , =GOOGLEFINANCE("KRX:005930","high52")
    - 가져오기 URL 형태: https://docs.google.com/spreadsheets/d/<ID>/gviz/tq?tqx=out:csv&sheet=<탭>
• KRX 개별종목의 한국 특화 지표(외국인/수급/배당성향/컨센서스)는 GF가 제공하지 않음.
• GF가 안정적으로 잘 주는 것 = 글로벌 지수 · 환율 · 범용 시세(가격/등락/52주/거래량).
• 4회/일 갱신 조건에서 GF 사용 = 가능(OK). 단 역할을 글로벌·보조로 한정 권장.


────────────────────────────────────────────────────────────
2. 구글파이낸스로 가져올 지표  (GOOGLEFINANCE 함수 / 게시 CSV)
────────────────────────────────────────────────────────────
[글로벌 지수]  ← 대시보드 상단 보조 표시
  • 나스닥        =GOOGLEFINANCE(".IXIC")
  • 다우          =GOOGLEFINANCE(".DJI")
  • S&P500        =GOOGLEFINANCE(".INX")
  • (속성) price / changepct

[환율]  ← 대시보드 환율 카드(현재 data.go.kr 사용 → GF로 보강/교차검증 가능)
  • USD/KRW       =GOOGLEFINANCE("CURRENCY:USDKRW")
  • 기타          EURKRW, JPYKRW, CNYKRW

[종목 시세 — 교차검증/백업용 (KRX)]
  • 현재가        =GOOGLEFINANCE("KRX:종목코드","price")
  • 등락률        ...,"changepct"
  • 시가/고가/저가 ...,"priceopen" / "high" / "low"
  • 거래량        ...,"volume"
  • 52주 최고/최저 ...,"high52" / "low52"
  • 통화          ...,"currency"
  ※ KRX는 GF에서 marketcap·pe가 N/A이거나 지연되는 경우 많음 → 네이버를 우선 사용,
     GF는 "네이버 실패 시 폴백" 또는 "가격 교차검증"으로만 활용.

[미국 상장 비교군 (선택)]
  • 해외 동종/ADR 가격·PER 등 — 필요 시 GF로


────────────────────────────────────────────────────────────
3. 네이버증권으로 가져올 지표  (주력 — 한국 종목 상세)
────────────────────────────────────────────────────────────
[A] 모바일 통합 API (JSON · 1회 호출로 대부분 수급/밸류 확보)
    GET https://m.stock.naver.com/api/stock/{종목코드}/integration
    (헤더: User-Agent 브라우저값, Referer: https://m.stock.naver.com/)
  • 현재가/전일/시가/고가/저가         (totalInfos)
  • 거래량/거래대금/시가총액
  • 외국인 보유율(외인소진율)          ← GF·DART에 없음, 네이버 전용
  • PER / EPS / PBR / BPS              ← PBR/BPS 실측(현 DART 근사 대체)
  • 추정 PER / 추정 EPS (포워드 컨센서스)
  • 배당수익률 / 주당배당금            (DART와 교차검증)
  • 52주 최고 / 52주 최저
  • 동일업종 비교종목 / 업종 PER       (industryCompareInfo)
  • 목표주가 컨센서스                  (consensusInfo)
  • 애널리스트 리포트 목록             (researches)
  • IR 일정 / 주주총회 일정            (irScheduleInfo, shareholdersMeetingInfo)

[B] 수급(외국인·기관·개인 순매수) 일별
    • 최근 5일: 위 통합 API의 dealTrendInfos
        foreignerPureBuyQuant / organPureBuyQuant / individualPureBuyQuant / foreignerHoldRatio
    • 20일+: https://finance.naver.com/item/frgn.naver?code={종목코드}  (HTML 표 파싱)

[C] OHLC 가격 이력 (일/주/월 + 일별 외국인소진율)
    GET https://api.finance.naver.com/siseJson.naver?symbol={코드}&requestType=1
        &startTime=YYYYMMDD&endTime=YYYYMMDD&timeframe=day|week|month
    • [날짜,시가,고가,저가,종가,거래량,외국인소진율] 배열
    • 캔들차트·52주범위·수익률 비교에 사용 (현재 data.go.kr+클라이언트 집계를 단순화 가능)


────────────────────────────────────────────────────────────
4. 기존 합법 공공 출처 (유지 — 변경 없음)
────────────────────────────────────────────────────────────
[DART OpenAPI]  — 재무의 원천(가장 신뢰도 높음)
  • 재무제표: 매출/영업이익/순이익/자본총계/부채총계 (fnlttSinglAcntAll)
  • 주요지표: ROE 등 (fnlttSinglIndx)
  • 배당: 주당배당금/배당성향/배당수익률 (alotMatter)
  • → 5개년 손익 도넛, 부채비율·수익성 추이, 성장성(YoY)

[공공데이터포털 (data.go.kr)]
  • 전체시세 스냅샷 → 거래량/시가총액 상위 랭킹, 검색 인덱스
  • 코스피/코스닥 지수

────────────────────────────────────────────────────────────
6. 생략 (구현 까다로움 — 사용자 합의)
────────────────────────────────────────────────────────────
• 유동주식수 / 유동비율(free-float) — 네이버 메인 HTML 부분 파싱 필요, 신뢰도 낮음 → 생략
    (대안: 상장주식수 = 시가총액/현재가 추정값은 이미 표시 중)
• 경제 캘린더(미국 CPI·기준금리 등) — 증권 외 별도 소스 필요 → 생략(추후)
• 섹터(업종) 지수 차트 — 업종코드 매핑 복잡 → 생략(추후)


────────────────────────────────────────────────────────────
7. 도입 시 주의 (네이버)
────────────────────────────────────────────────────────────
• 네이버 금융은 비공식 API → 응답 구조가 예고 없이 바뀔 수 있음. 방어적 파싱 + 폴백 필수.
• ToS상 상용 재배포 제한. 취미·개인용 + 저빈도(4회/일) 수준에서 현실적 리스크 낮음.
  (DART·공공데이터는 합법 공개 출처라 성격이 다름.)
• 호출 시 User-Agent/Referer 헤더 필요, 종목 루프에 딜레이.

[구현 제안 — 합의 시]
  api/cron-naver.js  : 유니버스 종목별 통합 API → KV(fr_naver_{code})에
                       외국인보유율·수급(일별)·PBR/BPS·추정치·컨센서스 적재
  api/cron-gf.js     : 게시된 구글시트 CSV fetch → KV(fr_global)에 지수·환율 적재
  vercel.json crons  : 위 4개 시각으로 스케줄 (기존 cron-market/cron-fin도 함께)
  프론트            : 외국인보유율·수급 차트·정확 PBR·글로벌 지수를 실데이터로 교체
